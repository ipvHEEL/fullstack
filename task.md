Твоя первая задача: "Система комментариев для Items"
В шаблоне уже есть модель Item ( CRUD для предметов/товаров). Твоя задача — расширить функционал, добавив возможность пользователям оставлять комментарии к этим предметам.
Требования к бэкенду:
Модель данных:
Создай новую модель Comment (таблица в БД).
Поля: id, content (текст), created_at, updated_at.
Связи: user_id (кто написал, ForeignKey на User), item_id (к чему написано, ForeignKey на Item).
Важно: Не забудь про каскадное удаление (если удалят Item, комментарии тоже должны уйти, или наоборот — подумай над логикой).
API Endpoints (REST):
POST /api/v1/items/{item_id}/comments — создать комментарий.
GET /api/v1/items/{item_id}/comments — получить список комментариев к предмету (с пагинацией: skip, limit).
PUT /api/v1/comments/{comment_id} — редактировать свой комментарий.
DELETE /api/v1/comments/{comment_id} — удалить комментарий.
Бизнес-логика и валидация:
Только авторизованные пользователи могут создавать комментарии.
Редактировать и удалять комментарий может только его автор или суперпользователь (admin).
Текст комментария не должен быть пустым (валидация через Pydantic).
При получении списка комментариев, в ответе должна быть информация об авторе (хотя бы full_name или email).
Миграции:
Используй Alembic, чтобы создать миграцию для новой таблицы. Не создавай таблицы вручную через create_all.
Стиль кода:
Старайся следовать архитектуре проекта (схема crud, models, schemas, api).
Используй существующие зависимости для авторизации (Depends(get_current_user) и т.д.).
Бонус (по желанию, но плюс на ревью):
Напиши хотя бы один интеграционный тест для создания комментария.
Добавь проверку, чтобы нельзя было создать комментарий к несуществующему item_id.
